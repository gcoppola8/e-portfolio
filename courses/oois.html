Object-oriented Information Systems

Information systems are responsible for collecting, storing, processing and providing knowledge about information from a range of sources. Information systems are an integrated set of components that bring together hardware, software, networking, database storage as well as the human process that may be involved. These systems are integral to the products and services that we rely on as a society and as such it is imperative that they are well-designed and developed. Object oriented design and development provides a structured approach to creating sustainable and manageable solutions. The module examines the different elements of an information system from design through to processing and storage. You will explore the fundamental concepts behind object-oriented development using UML as a tool for analysis and design. Further to this, we will make use of Python to provide concrete experience of object-oriented development.

In this module we will:

  1. Contextualise the role of information systems.
  2. Understand object-oriented analysis and design.
  3. Engage with the issues of professionalism (including coding conventions and good practices).
  4. Develop object-oriented software using Python and Java.
  5. Create GUIs using Python.
  6. Introduce relational algebra and set theory.
  7. Contextualise database concepts, design and implementation.
  8. Introduce SQL development.
  9. Develop web applications using Python.
  10. Explore the integration of information system components into business processes.
  11. Introduce big data technologies.

Unit 12 - Emerging trends in Information Systems

Big data analytics

Big data is a set of technologies that allow to gather, store and analyze a large amount of data. It allows company to improve the performance of their systems, but also to analyze the data and find insights to drive their decision. Some advantages of using big data technologies are the capacity to store big amount of data and scale it when is needed, that data don't need to be consistent, indeed it is possible to store unomogeneous type of data, and that data is highly available, so the storage system in its complexity can bear failure of single machines without compromise its functions. One very popular tool regarding data storage is Hadoop, that nowadays shares the market with other solutions like MongoDb, AWS S3 and Azure Data Lake. These tools are used because traditional relational solutions aren't capable to keep up the large amount of data and high availability requirements needed. Once data are stored, they can be processed, each company based on their objectives is going to process data in different ways, to run research on them, categorize them, sell them etc. And as last trend in big data there is the visualization of these data. That means to give to the found insights a good presentation outlook, that can be used in charts, reports including non technical employee who are not familiar with more technical aspects of these technologies. Two very popular solutions in this sphere are Tableau and PowerBI(from Microsoft). I believe that this is a big trend nowadays because storage, processing and visualization of data have all a big potential for new developments.
